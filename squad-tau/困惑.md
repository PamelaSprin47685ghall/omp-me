## 和 Debug 脚本的区别

Debug 脚本不做 inject，它直接 click R1 authoring 检查 textarea。而 test 05 在 inject 之前先做了两次 sidebar click（flow-node → R1 authoring）。**也许第二次 click R1 authoring 没有生效？** test 05 的 click 序列是：

## 最重要的盲点

Test 05 在 inject 之后用 `waitForFunction` 检查 `el.textContent.includes('Planning')`，这个 text 来自 delta 事件。**我仍然不知道 element 是否存在**——可能 element 渲染了但 delta 没到达（delta 事件的 CustomEvent listener 没注册等）。

## 应该做的最少侵入诊断

修改 test 05 的 `waitForFunction` 让它在 fail 时提供诊断信息，而不是简单返回 false。例如：

```javascript
await page.waitForFunction(() => {
    const el = document.querySelector('agent-message');
    const root = el?.shadowRoot;
    const details = root?.querySelector('.thinking-section details');
    // 如果 element 存在但 text 没有，返回具体原因说明
    if (!el) return 'NO_EL';
    if (!root) return 'NO_SHADOW';
    if (!details) return 'NO_DETAILS';
    if (!details.open) return 'NOT_OPEN';
    if (!el.textContent || !el.textContent.includes('Planning')) 
        return 'NO_TEXT:' + (el.textContent || '').substring(0, 100);
    return true;
}, { timeout: T });
```

这样 timeout 时可以看到具体原因。但这个修改需要保持 test 不 throw。
