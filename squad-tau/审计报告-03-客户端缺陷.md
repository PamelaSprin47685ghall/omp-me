# Squad-Tau 功能审计报告 — 客户端缺陷

## 高 (High)

### C-H-01: `InputSection` 输入框禁用逻辑检查 `.status` 而非 `.phase`

**文件**: `client/components/MainContent.jsx:64-67`  
**实际行为**:
```js
const reason = ['completed', 'aborted', 'error'].includes(activeSession.status) 
    ? activeSession.status : null;
```
此逻辑依赖 `session:end` 事件设置 `.status` 为终止值。但 `handleSessionStateChange`（session-reducer.js）只更新 `.phase`，不更新 `.status`。如果服务端发送 `session:state { phase: 'completed' }` 而 `session:end` 事件丢失或延迟，输入框不会禁用，用户可向已结束的 session 发消息。

**PRD 要求**: §3.5 "已结束的 session 拒绝接收用户消息，Web UI 会禁用输入框"

**更稳健的做法**: 同时检查 `.status` 和 `.phase`。

### C-H-02: Sidebar `getStatusIcon('active')` 回退到 HELP 图标

**文件**: `client/components/Sidebar.jsx:16-23`  
**实际行为**: `getStatusIcon` 的 switch 没有 `'active'` case，`session.status === 'active'`（初始值）回退到 `IconNames.HELP`。PRD §4.4 图标指南要求 `active` 对应"进行中/刷新"类图标。

**影响**: SessionTree 中所有活跃会话右侧显示问号图标而非刷新图标，视觉语义错误。

## 中 (Medium)

### C-M-01: `useAutoScroll` 违反 React Hooks 规则

**文件**: `client/hooks/useAutoScroll.js`  
**实际行为**: `useEffect` 的依赖数组通过 spread 操作符 `...deps` 传入，React Hooks 规范要求依赖数组是写死的字面量数组。spread 方式在某些 React 版本可能导致 lint 警告，且deps 顺序变化时行为不可预测。

**功能影响**: 当前使用可能正常，但维护风险高。

### C-M-02: `SessionTree.updateNode` 直接突变状态对象

**文件**: `client/components/SessionTree.jsx:28-33`  
**实际行为**:
```js
const updateNode = useCallback((nodePath, isExpanded) => {
    setContents(curr => {
        const updated = [...curr];
        const node = getNodeAtPath(updated, nodePath);
        if (node) node.isExpanded = isExpanded;  // 直接突变
        return updated;
    });
}, []);
```
浅拷贝后直接修改节点对象引用。虽然外层数组已复制，但内层 node 对象是共享引用。React 的 reconciliation 可能检测不到变化，导致 DOM 不更新。

### C-M-03: `SessionTree.getNodeAtPath` 缺少 childNodes 防御

**文件**: `client/components/SessionTree.jsx:24-28`  
**实际行为**: 
```js
function getNodeAtPath(nodes, path) {
    let current = nodes[path[0]];
    for (let i = 1; i < path.length; i++) {
        current = current.childNodes[path[i]];
    }
    return current;
}
```
如果某个中间节点没有 `childNodes` 属性，`current.childNodes` 为 `undefined`，下一轮 `undefined[path[i]]` 抛 TypeError。Blueprint Tree 在折叠叶子节点时可能产生此场景。

### C-M-04: `App.useAppEventHandlers` 忽略 `error` 类型事件

**文件**: `client/App.jsx:18-28`  
**实际行为**: 事件路由只处理 `squad:`、`session:`、`model_pool:` 前缀。服务端发送的 `{ type: 'error', payload: { message } }` 事件不匹配任何前缀，被静默丢弃。PRD §5.7 定义了 `error` 事件用于通知客户端操作失败。

**影响**: 用户向已结束 session 发消息时，服务端返回的 `{ type: 'error', message: 'Session not active' }` 被客户端忽略，用户无任何反馈。

## 低 (Low)

### C-L-01: `useWebSocket` 无最大重连次数限制

**文件**: `client/hooks/useWebSocket.js`  
**实际行为**: 指数退避到 30s 后保持 30s 间隔无限重连。PRD §4.10 只说"指数退避 1s→30s"，未定义最大次数。但永远重连可能导致电池/带宽浪费，特别是在服务已停止的场景下。

### C-L-02: `useDarkMode` 卸载时不移除 DARK class

**文件**: `client/hooks/useDarkMode.js`  
**实际行为**: `useEffect` 添加 `Classes.DARK` 到 document，但 `cleanup` 函数未移除。如果组件卸载并重新挂载（如 React 18 StrictMode 双挂载），class 不会被清理。对单页应用影响极小。

### C-L-03: `squad:complete` 结果存储但从未显示

**文件**: `client/hooks/useSquadState.js`  
**实际行为**: `SQUAD_COMPLETE` action 将 `payload.results` 存入 state，但没有任何组件读取或渲染这些结果。完成摘要只在 `squad:complete` 事件被接收时显示在 Header/StatusBar 中，但 UI 没有专门的结果展示区域。
