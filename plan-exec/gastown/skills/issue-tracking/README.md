# Issue Tracking Skill

Master-level issue tracking adapted from [Gas Town](https://github.com/steveyegge/gastown) by Steve Yegge.

## Purpose

Track beads, review rounds, TDD cycles, gate votes, and conflict repairs as first-class issues with immutable audit trail.

## Process Flow

1. Create issues for each bead in the DAG
2. Record review-loop rounds with deliverable snapshots
3. Log TDD cycles with red-green-refactor phases
4. Capture gate votes with voter ID, confidence, and reasoning
5. Document conflict repairs with fix agent, files modified, test results
6. Maintain immutable audit trail for dispute resolution

## Integration

- **Input from:** All Gas Town processes
- **Output to:** Patrol reports, attribution records, audit logs
- **Used by:** All processes (implicit)
